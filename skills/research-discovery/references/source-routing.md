# Source Routing

## Literature

- Biomedical or clinical topics: PubMed first, then Semantic Scholar, CrossRef, journal pages, and web search.
- CS or engineering methods: arXiv first, then Semantic Scholar, CrossRef, GitHub, Papers with Code, and web search.
- Mixed medical AI topics: PubMed plus Semantic Scholar plus arXiv.
- Clinical guidelines: official society pages, journal publications, PubMed records, and DOI pages.

## Datasets

Prefer source-of-truth records in this order:

1. Dataset paper DOI or PubMed record.
2. Official dataset repository, challenge page, or institutional page.
3. Zenodo, Figshare, TCIA, PhysioNet, Hugging Face, Kaggle.
4. Author GitHub release or README.
5. Third-party mirror only when clearly marked as risky.

## Code and Baselines

Prefer:

1. Official paper repository.
2. Author-maintained implementation.
3. Benchmark repository with documented data preparation.
4. High-quality third-party implementation.

Record whether the repo has a license, train/eval scripts, checkpoints, dataset instructions, recent commits, and reproducible commands.

## Zotero and Local Sources

Use Zotero only when the user asks to use a local library or when a referenced source is already known to be in the user's library. Zotero records must keep Zotero item keys separate from BibTeX keys.

## Weak Sources

Blog posts, social posts, mirror datasets, and unverified repos can seed leads, but they cannot support a route without a stable source behind them.
